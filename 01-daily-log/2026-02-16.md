## 오늘 한 것
- 강의: Java - 메서드
- 실습: 메서드 정의, 반환 타입, 메서드 호출과 값 전달, 메서드와 형변환, 메서드 오버로딩
- 정리:
1. 메서드가 필요한 이유
같은 로직을 여러 번 쓰면 중복이 생김 → 수정/확장 시 다 고쳐야 함
메서드로 “기능”을 묶으면 
- 재사용 가능
- main()이 “목차”처럼 읽힘(구조가 보임)
- 수정 범위가 메서드 내부로 좁아짐(유지보수 쉬움)

2. 메서드 기본 형태(정의/호출)
정의(선언부 + 본문)
public static 반환타입 메서드이름(매개변수...) {
  // 실행코드
  return 반환값; // 반환타입이 void면 생략 가능
}
호출
결과변수 = 메서드이름(인수...);
메서드이름(인수...); // 반환값 안 쓰면 이렇게 호출만 가능

인수(Argument): 호출할 때 넘기는 값

매개변수(Parameter): 메서드 정의에 있는 변수(받는 그릇)

3. 반환 타입(return type)

반환 타입이 있으면: 모든 실행 경로에서 반드시 return 값이 있어야 함
(if만 있고 else가 없으면 컴파일 오류 “missing return statement” 가능)
반환 타입이 없으면(void): return 생략 가능(자바가 마지막에 자동 return 처리)
return의 의미
return을 만나면 즉시 메서드 종료
void에서도 “조기 종료”하려면 return; 사용 가능

4. 값 전달의 핵심 규칙(진짜 중요)
자바는 항상 “값을 복사해서” 전달한다
기본형(int 등): 값 자체 복사
참조형(배열/객체): “주소값(참조값)”을 복사 (이건 뒤에서 더 자세히)
그래서 메서드 안에서 매개변수를 바꿔도 원래 변수는 안 바뀜
→ 바꾸고 싶으면 반환값을 받아서 대입해야 함.

예)
changeNumber(num1); → num1 그대로
num1 = changeNumber(num1); → num1 변경됨(반환값 반영)

5. 메서드 호출과 형변환
메서드 호출도 결국 “대입”이므로 형변환 규칙 그대로
큰 범위로는 자동 변환 가능: int → long → double
double → int 처럼 줄어드는 변환은 명시적 캐스팅 필요

6. 메서드 오버로딩(Overloading)
같은 이름 + 다른 매개변수(타입/개수/순서)면 여러 개 정의 가능

가능:
add(int, int)
add(int, int, int)
add(double, double)
myMethod(int, double) vs myMethod(double, int)

불가능:
매개변수 동일하고 반환 타입만 다른 것
int add(int,int) / double add(int,int) → 컴파일 오류
메서드 시그니처(signature)
메서드 이름 + 매개변수 타입(순서 포함)
자바는 이걸로 어떤 메서드를 호출할지 결정
오버로딩 선택 우선순위(감각만)
“가장 정확히 맞는 타입” 우선
없으면 “자동 형변환 가능한 쪽”을 찾음
