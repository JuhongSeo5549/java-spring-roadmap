## 오늘 한 것

========================================
Object와 다형성 / toString 정리
========================================


[1] Object를 활용한 다형성
----------------------------------------
Object는 자바의 모든 클래스의 최상위 부모 클래스이다.
즉, 모든 객체를 담을 수 있는 타입이다.

쉽게 말하면
- String 객체도 담을 수 있고
- Integer 객체도 담을 수 있고
- 사용자가 만든 클래스 객체도 담을 수 있다.

예)
Object obj1 = "hello";
Object obj2 = 10;
Object obj3 = new Dog();

정리하면,
Object는 "모든 객체를 참조할 수 있는 공통 부모"이다.


[2] Object를 사용한 다형성의 한계
----------------------------------------
Object는 모든 객체를 담을 수는 있지만,
각 객체의 고유 기능까지 바로 사용할 수는 없다.

이유는 Object 타입으로 참조하면
컴파일 시점에는 Object 클래스에 정의된 메서드만 사용할 수 있기 때문이다.

예)
Object obj = new Dog();
// obj.bark();  -> 호출 불가

Dog의 bark() 같은 메서드를 호출하려면
반드시 다운캐스팅이 필요하다.

예)
Dog dog = (Dog) obj;
dog.bark();

즉,
- Object는 모든 객체를 담을 수 있다.
- 하지만 모든 자식 클래스의 메서드를 알고 있지는 않다.
- 그래서 자식 고유 기능을 사용하려면 다운캐스팅이 필요하다.


[3] 다형성을 제대로 활용하려면
----------------------------------------
다형성을 제대로 활용하려면
"다형적 참조 + 메서드 오버라이딩"이 함께 필요하다.

예)
Animal animal = new Dog();
animal.sound();

이 경우 변수 타입은 Animal이지만,
실행되는 메서드는 Dog가 재정의한 sound()이다.

이런 방식이면 다운캐스팅 없이도
부모 타입으로 자식의 기능을 유연하게 사용할 수 있다.

결론적으로,
Object를 사용한 다형성은 "참조"는 가능하지만
"활용"에는 한계가 있다.

왜냐하면 Object에는 자식들의 고유 메서드가 정의되어 있지 않기 때문이다.


[4] Object 배열
----------------------------------------
Object는 모든 객체를 담을 수 있으므로
Object[] 배열도 모든 객체를 저장할 수 있다.

예)
Object[] arr = new Object[3];
arr[0] = "문자열";
arr[1] = 100;
arr[2] = new Dog();

특징
- 서로 다른 타입의 객체를 한 배열에 저장할 수 있다.
- 모든 객체를 담을 수 있다.
- 꺼내서 사용할 때는 필요한 타입으로 다운캐스팅이 필요할 수 있다.


[5] toString()
----------------------------------------
toString()은 Object 클래스에 정의된 메서드이다.
객체 정보를 문자열 형태로 반환할 때 사용한다.

그래서 디버깅, 로그 확인, 객체 출력 시 매우 자주 사용된다.

모든 클래스는 Object를 상속받기 때문에
toString()도 기본적으로 사용할 수 있다.


[6] Object의 기본 toString()
----------------------------------------
Object가 기본 제공하는 toString()은 아래와 같다.

public String toString() {
    return getClass().getName() + "@" + Integer.toHexString(hashCode());
}

기본적으로 반환되는 값은 다음과 같다.
- 클래스 이름
- @
- 해시코드의 16진수 값

예)
com.example.Dog@1a2b3c

이 값은 보통
"객체의 클래스 정보 + 해시값" 형태라고 이해하면 된다.

참고
- 흔히 참조값처럼 보인다고 표현하지만
  정확히는 hashCode() 결과를 16진수 문자열로 보여주는 것이다.


[7] println()과 toString() 결과가 같은 이유
----------------------------------------
System.out.println()에 객체를 넣으면
내부적으로 toString()이 호출된다.

예)
Dog dog = new Dog();

System.out.println(dog);
System.out.println(dog.toString());

위 두 코드는 결과가 같다.

이유는 println()이 객체를 출력할 때
자동으로 toString()을 호출하기 때문이다.

즉, 객체를 출력할 때는
toString()을 직접 적지 않아도 된다.


[8] toString() 오버라이딩
----------------------------------------
기본 toString()은 클래스명과 해시값만 보여주기 때문에
객체의 실제 상태를 파악하기에는 부족한 경우가 많다.

그래서 보통은 toString()을 오버라이딩해서
객체의 중요한 정보를 문자열로 직접 반환하도록 만든다.

예)
public class Dog {
    String name;
    int age;

    public Dog(String name, int age) {
        this.name = name;
        this.age = age;
    }

    @Override
    public String toString() {
        return "Dog{name='" + name + "', age=" + age + "}";
    }
}

출력 예)
Dog{name='멍멍이', age=3}

오버라이딩하는 이유
- 객체 상태를 쉽게 확인할 수 있다.
- 디버깅이 편해진다.
- 로그 확인이 쉬워진다.
- 출력 결과가 직관적이다.


[9] 핵심 요약
----------------------------------------
1. Object는 모든 객체의 부모이므로 모든 객체를 참조할 수 있다.

2. 하지만 Object 타입으로는 자식 고유 메서드를 바로 호출할 수 없다.

3. 자식 기능을 사용하려면 다운캐스팅이 필요하다.

4. 그래서 Object만으로는 다형성을 완전하게 활용하기 어렵다.

5. 다형성을 제대로 활용하려면
   "다형적 참조 + 메서드 오버라이딩"이 함께 필요하다.

6. Object[] 배열은 모든 객체를 담을 수 있다.

7. toString()은 객체 정보를 문자열로 반환하는 메서드이다.

8. println()은 내부적으로 toString()을 호출한다.

9. 기본 toString()은 클래스명 + 해시값 형태이다.

10. 보통은 객체 상태를 잘 보여주도록 toString()을 오버라이딩해서 사용한다.


[10] 한 줄 정리
----------------------------------------
Object는 모든 객체를 담을 수 있는 공통 부모이지만,
자식 고유 기능까지 직접 쓸 수는 없어서
실전 다형성 활용에는 한계가 있다.
그리고 toString()은 객체 정보를 보기 좋게 출력하기 위해 자주 오버라이딩한다.
