## 오늘 한 것
- 강의: Java - 추상클래스, 인터페이스
- 실습: 추상클래스와 추상메서드, 인터페이스
- 정리:
■ 추상 클래스(Abstract Class)

하나 이상의 추상 메서드를 가질 수 있는 클래스

인스턴스 생성 불가

상속을 통해서만 사용

자식 클래스는 추상 메서드를 반드시 오버라이딩 해야 함

효과

의미 없는 객체 생성을 방지

자식 클래스에서 필수 메서드 구현을 강제

■ 순수 추상 클래스(Pure Abstract Class)

모든 메서드가 추상 메서드인 추상 클래스

인스턴스 생성 불가

상속 시 모든 메서드 구현 필수

주 용도: 다형성

특징상 인터페이스와 매우 유사함

■ 인터페이스(Interface)

메서드 이름만 정의된 설계도(규격)

모든 메서드는 public abstract

멤버 변수는 public static final(상수)

인스턴스 생성 불가

다중 구현 가능

■ 상속 vs 구현

클래스 → 클래스 : 상속(extends)

클래스 → 인터페이스 : 구현(implements)

이유

상속은 기능을 물려받음

인터페이스는 기능이 없고, 자식이 직접 만들어야 하므로 구현이라 표현

■ 인터페이스를 사용하는 이유

강한 제약 제공

구현 클래스가 반드시 모든 메서드를 구현해야 함

설계 규칙을 깨기 어렵다

다중 구현 가능

여러 인터페이스를 동시에 구현 가능

구조가 명확한 좋은 설계 유도

■ 클래스 다중 상속이 불가능한 이유

부모가 여러 개일 경우 어떤 부모의 메서드를 사용할지 애매해짐(다이아몬드 문제)

구조 복잡도 증가

■ 인터페이스 다중 구현이 가능한 이유

인터페이스는 구현 코드가 없음

실제 구현은 자식 클래스에서 하나만 작성

■ 핵심 요약

추상 클래스 : 공통 기능 + 일부 구현 강제

인터페이스 : 규격만 제공 + 전부 구현 강제

좋은 프로그램은 제약이 있는 프로그램이다
