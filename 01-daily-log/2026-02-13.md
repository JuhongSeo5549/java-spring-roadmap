## 오늘 한 것
- 강의: Java - 스코프,형변환
- 실습: 지역 변수와 스코프. 스코프 존재 이유, 자동형변환, 명시적 형변환, 계산과 형변환
- 정리:
1. 스코프(Scope)
변수가 “접근 가능한 범위”를 말한다.
지역 변수는 선언된 코드 블록 {} 안에서만 살아있고, 블록을 벗어나면 제거되어 접근 불가이다.
바깥 블록 변수는 안쪽 블록에서 접근 가능하지만, 안쪽 블록 변수는 바깥에서 접근 불가이다.

2. 스코프가 존재하는 이유
변수는 필요한 범위에서만 쓰는 것이 좋다.
스코프를 좁히면 불필요한 메모리 사용을 줄이고 유지보수 시 신경 써야 할 변수가 줄어 코드가 단순해진다.
for문은 카운터 변수를 for 블록 안에 가둘 수 있어(스코프 축소) while문보다 스코프/유지보수 관점에서 유리한 경우가 많다.

3. 형변환(Casting) 기본 개념
타입을 다른 타입으로 바꾸는 것(대입/연산에서 발생).
작은 범위 -> 큰 범위: 안전(값 손실 가능성 낮음), 큰 범위 -> 작은 범위: 위험(값 손실 가능)
숫자 범위 관계(대표) : int < long < double

4. 자동 형변환(묵시적 형변환)
작은 타입 값을 큰 타입 변수에 넣을 때 자동으로 변환된다.
예: int -> long, int -> double, long -> double

5. 명시적 형변환(강제 형변환)
큰 타입 값을 작은 타입에 넣을 때는 컴파일 에러가 나므로, 개발자가 형변환을 “명시”해야 한다.

5.1 명시적 형변환 시 대표 위험
1.소수점 버림(double -> int)
2.오버플로우(범위 초과 시 엉뚱한 값으로 바뀜)
중요: 오버플로우 결과를 계산하려 하지 말고, 애초에 더 큰 타입을 사용해 발생 자체를 막는다.

6. 계산과 형변환 규칙
같은 타입끼리 계산하면 결과도 같은 타입이다.
서로 다른 타입끼리 계산하면 큰 타입으로 자동 형변환된다.

6.1 추가 핵심:
정수끼리 나눗셈(int / int)은 결과도 정수라 소수점이 버려진다.
소수 결과가 필요하면 나눗셈 전에 double로 형변환이 필요하다.
